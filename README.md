# 💻 [2026-01] 나만의 C언어 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 야구 타자 성적 관리 시스템 

<br>


## 2. 시나리오 (5~10줄)
* 이 프로그램은 야구 타자의 타격 성적을 입력받아 계산하고 관리하는 콘솔 프로그램입니다.
사용자는 타자의 이니셜, 경기 수, 타수, 안타 수, 홈런 수, 지난 시즌 타율 등을 입력할 수 있습니다.
시스템은 입력된 데이터를 바탕으로 현재 타율을 계산하고, 지난 시즌 타율과의 차이를 출력합니다.
이후 프로그램이 확장되면 현재 타율에 따라 타자의 등급을 판정할 수 있습니다.
또한 홈런 수와 타율을 함께 고려하여 강타자 여부를 판단하는 기능도 추가할 수 있습니다.
최종적으로는 메뉴를 통해 원하는 기능을 선택하고, 여러 명의 타자 정보를 배열로 저장하여 관리할 수 있도록 발전시킬 계획입니다.
즉, 이 프로그램은 수업 시간에 배운 성적 관리 프로그램의 구조를 바탕으로, 야구라는 저의 관심사를 접목해 만든 타자 성적 관리 시스템입니다. 

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
1. 타자 정보 입력
2. 현재 타율 계산
3. 타자 등급 판정
4. 강타자 여부 확인
5. 타자 성적 확인
6. 프로그램 종료

## 🟦 [1차 과제: V1.0] 기초 뼈대 구축 (입출력과 변수)
### **✨V1.0 개발 내용:**
* 사용자로부터 타자의 이니셜, 경기 수, 타수, 안타 수, 홈런 수, 지난 시즌 타율을 입력받습니다.
* 입력받은 데이터를 바탕으로 현재 타율(안타 ÷ 타수)을 계산합니다.
* 또한 현재 타율과 지난 시즌 타율의 차이를 계산하여 출력합니다.
* 결과 화면에서는 타자의 기본 정보와 함께 계산된 현재 타율과 타율 차이를 출력합니다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** C언어 4단계 과제 구조(입출력 → 조건문 → 반복문/메뉴 → 배열/함수)에 맞는 야구 타자 성적 관리 프로그램을 기획해 달라고 요청함.
    * **적용 내용:** 성적 관리 프로그램의 구조를 참고하여, 타자 정보 입력 → 현재 타율 계산 → 타자 등급 판정 → 메뉴 시스템 → 다수 선수 관리로 이어지는 단계별 확장 구조를 설계함.
      
 2. **내용 2**
    * **프롬프트 요약:** 1차 과제 조건(변수 5개 이상, 자료형 3개 이상)을 만족하는 타자 성적 프로그램의 변수 구조와 출력 형식을 설계해 달라고 요청함.
    * **적용 내용:** 문자형(char - 타자 이니셜), 정수형(int - 경기 수, 타수, 안타 수, 홈런 수), 실수형(float - 지난 시즌 타율, 현재 타율, 타율 차이)으로 자료형을 구분하여 변수 구조를 설계함.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 타율을 계산할 때 안타 수와 타수를 모두 정수형으로 사용하면 소수점 아래 값이 정확히 표현되지 않는 문제가 발생함.
     * **해결:** 실수형 변수 f_hit, f_at_bat를 추가하여 안타 수와 타수를 실수형으로 저장한 뒤, 현재 타율을 실수형 나눗셈으로 계산하도록 수정함.
  2. **문제 2:** 실수형으로 계산한 타율과 타율 차이를 출력할 때 소수점 자리 수가 일정하지 않아 결과 화면이 깔끔하지 않게 보일 수 있었음.
     * **해결:** printf에서 %.3f 형식을 사용하여 소수점 셋째 자리까지 출력하도록 수정함.
  3. **문제 3:** scanf("%c", &initial);로 문자를 입력받을 때 엔터가 함께 입력되는 문제가 발생함..
     * **해결:** %c 앞에 공백을 넣은 scanf(" %c", &initial); 형태로 수정하여 문제를 해결함.

## 🟩 [2차 과제: V2.0] 조건 분기 적용 (조건문) - 향후 작성 예정
### **✨2차 과제 업데이트 내용:**
  * V1.0에서 계산한 현재 타율을 기준으로 타자 등급을 판정하는 기능을 추가한다.
  * 'if ~ else if ~ else' 다중 조건문을 사용하여 타율 구간에 따라 S급, A급, B급, C급, D급 타자로 분류한다.
  * 관계 연산자 '>=', '<'를 활용하여 현재 타율이 어느 구간에 속하는지 비교한다.
  * 중첩 조건문을 사용하여 A급 이상 타자 중에서 홈런 수가 30개 이상인지 추가로 판정한다.
  * 논리 연산자 '&&'를 활용하여 현재 타율이 0.300 이상이고 홈런 수가 30개 이상인 경우를 강타자로 판정한다.
    
### **🤖 AI 파트너십 과정**
 1. **내용 1**
    * **프롬프트 요약:** 1차 과제에서 만든 야구 타자 성적 관리 시스템을 2차 과제 조건인 조건문 중심으로 확장하는 방법을 질문함. 
    * **적용 내용:** 기존 V1.0의 입력, 계산, 출력 구조는 유지하고, 현재 타율을 기준으로 타자 등급을 판정하는 다중 조건문 구조를 추가하기로 함.

 2. **내용 2**
    * **프롬프트 요악:** 다중 조건문, 중첩 조건문, 논리 연산자, 관계 연산자를 모두 포함하여 야구 타자 성적 관리 시스템을 확장하는 방법을 질문함.
    * **적용 내용:** 현재 타율 구간을 '>=', '<' 관계 연산자로 비교하여 등급을 판정하고, A급 이상 타자 중 홈런 수를 추가로 검사하는 중첩 조건문을 설계함. 또한 '&&' 논리 연산자를 사용하여 타율과 홈런 수를 동시에 만족하는 강타자 판정 기능을 추가하기로 함.
      
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** 타율 등급을 판정할 때 조건 순서를 낮은 기준부터 작성하면 높은 타율의 선수도 낮은 등급으로 먼저 판정될 수 있는 문제가 발생할 수 있음.
     * **해결:** 'if ~ else if ~ else' 다중 조건문을 사용할 때 가장 높은 기준인 0.330 이상부터 차례대로 검사하도록 조건 순서를 정리함.
  
  2. **문제 2:** 타율과 홈런 수를 함께 고려하는 조건을 따로 작성하면 코드가 길어지고 판정 기준이 명확하지 않을 수 있음.
     * **해결:** 논리 연산자 '&&'를 사용하여 '현재 타율 >= 0.300' 그리고 '홈런 수 >= 30'을 동시에 만족하는 경우를 강타자로 판정하도록 수정함.

  3. **문제 3:** 중첩 조건문을 사용할 때 바깥쪽 조건과 안쪽 조건의 역할이 헷갈릴 수 있었음.
     * **해결:** 바깥쪽 조건문에서는 현재 타율이 0.300 이상인지 먼저 확인하고, 그 안쪽 조건문에서 홈런 수가 30개 이상인지 추가로 검사하도록 구조를 분리함.
